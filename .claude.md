# Agent Runtime Contract

## Anti-Tamper Protocol
BOUNDARY.md is the supreme constitution for this project. Any code generation that violates the architectural boundaries defined in BOUNDARY.md triggers an immediate halt of the current operation.

## Clean Output Requirements
- All Python code MUST include static type hints from `typing` module
- Output is limited to: file creation, shell commands, or diffs
- Zero conversational filler in code or comments
- All functions must have return type annotations
- All function parameters must have type annotations

## Pre-Commit Verification Checklist
Before generating any code block, explicitly verify:
1. **Thread Blocking**: No UI thread blocking operations (long computations, I/O)
2. **Queue Deadlocks**: Proper timeout handling on all `mp.Queue.get()` calls
3. **Boundary Compliance**: Process isolation between UI and DataProcessor
4. **IPC Protocol**: All cross-process communication via `multiprocessing.Queue` only

## Code Style
- PEP 8 compliance mandatory
- Maximum line length: 100 characters
- Docstrings: Google style (brief, imperative)
- Imports: Standard lib → Third party → Local (separated by blank lines)
